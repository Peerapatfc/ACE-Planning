- Tag follows contact across all their conversations
- Filter conversation list by contact tag
- Admin library management: rename/delete, usage count
- Audit log on tag add/remove
- Max 10 tags per contact, tenant isolation

# Scenario รีวิว (demo)

> Demo script พร้อมใช้: `scripts/test-contact-tags.sh` (ต้องใช้ admin token — rename ต้อง `org:contact_tag:manage`, delete ต้อง `org:workspace:manage`)
> Endpoint: `GET/POST /contact-tags`, `PUT/DELETE /contact-tags/:id`, `POST/DELETE /contacts/:id/tags/:tagId`

## Scenario 1 — ผูก tag เข้า contact แล้วตามข้าม conversation
- **ขั้นตอน:** ที่ contact info panel ผูก tag "VIP" ให้ลูกค้า A → เปิด conversation อื่นของลูกค้า A คนเดียวกัน
- **คาดหวัง:** tag "VIP" แสดงในทุก conversation ของลูกค้า A อัตโนมัติ (tag ผูกกับ contact ไม่ใช่ conversation)

## Scenario 2 — สร้าง tag แบบ inline (typeahead)
- **ขั้นตอน:** พิมพ์ชื่อ tag ใหม่ในช่องค้นหา → เลือก "สร้าง ..."
- **คาดหวัง:** tag ถูกสร้างและผูกเข้า contact ในขั้นตอนเดียว

## Scenario 3 — library แยกจาก conversation tags
- **ขั้นตอน:** สร้าง contact tag ชื่อซ้ำกับ conversation tag ที่มีอยู่
- **คาดหวัง:** สร้างได้ ไม่ชนกัน (คนละ library) — จุดโชว์ที่เป็นเหตุผลของ ACE-2295 (ไม่ merge tag library)

## Scenario 4 — filter รายการแชทด้วย contact tag
- **ขั้นตอน:** เลือก filter หลาย tag พร้อมกัน → refresh หน้า
- **คาดหวัง:** กรองแบบ AND (ต้องมีครบทุก tag), และ filter คงอยู่ผ่าน URL param

## Scenario 5 — จัดการ library ตามสิทธิ์ (RBAC)
- **ขั้นตอน:** rename และ delete tag ด้วย token ของ agent / supervisor / admin
- **คาดหวัง:**
  - rename: admin ✓, supervisor ✓, agent → 403
  - delete: admin ✓, supervisor → 403, agent → 403
  - แสดง usage count (จำนวน contact ที่ใช้ tag นั้น)

## Scenario 6 — เพดาน 10 tags/contact
- **ขั้นตอน:** ผูก tag ให้ contact จนครบ 10 → พยายามผูกตัวที่ 11
- **คาดหวัง:** ตัวที่ 11 ถูกปฏิเสธพร้อมข้อความ

## Scenario 7 — audit log
- **ขั้นตอน:** add แล้ว remove tag ออกจาก contact
- **คาดหวัง:** มี log บันทึกทั้ง add และ remove (ใคร/เมื่อไหร่)

## Scenario 8 — tenant isolation
- **ขั้นตอน:** ใช้ token ของ workspace A เรียก/ผูก tag ของ workspace B
- **คาดหวัง:** มองไม่เห็น tag ข้าม workspace — ไม่มีข้อมูลรั่ว, ไม่ 500

## Scenario 9 — normalize ชื่อ tag (มี unit test ยืนยัน)
- **ขั้นตอน:** สร้าง tag ชื่อ " vip " (มี space, ตัวเล็ก) → ค้นหาด้วย "vip"
- **คาดหวัง:** ได้ tag ชื่อ "VIP" (trim + uppercase อัตโนมัติ), ค้นหาแบบ case-insensitive เจอ

## Scenario 10 — ชื่อซ้ำถูกปฏิเสธ (มี unit test ยืนยัน)
- **ขั้นตอน:** สร้าง tag ชื่อที่มีอยู่แล้ว และ rename tag ไปชนชื่อที่มีอยู่
- **คาดหวัง:** โดนปฏิเสธทั้ง 2 กรณี พร้อมข้อความชัดเจน (ไม่ 500)

## Scenario 11 — idempotent: กดซ้ำไม่พัง (มี unit test ยืนยัน)
- **ขั้นตอน:** assign tag เดิมซ้ำ 2 ครั้ง / remove tag ที่ถูก remove ไปแล้ว
- **คาดหวัง:** no-op เงียบ ๆ ไม่ error, ไม่เกิด event ซ้ำ (กันผู้ใช้กดรัว)

## Scenario 12 — realtime ข้ามจอ (มี unit test ยืนยัน publish event)
- **ขั้นตอน:** เปิด 2 จอ (agent 2 คน ดู contact เดียวกัน) → จอแรก assign/rename/delete tag
- **คาดหวัง:** จอที่สองเห็นการเปลี่ยนแปลงทันทีโดยไม่ต้อง refresh — จุดโชว์เด่นใน demo

## Scenario 13 — race: tag ถูกลบระหว่าง assign (มี unit test ยืนยัน)
- **ขั้นตอน:** admin ลบ tag จาก library ขณะที่อีกจอกำลังจะ assign tag นั้น
- **คาดหวัง:** ได้ error ชัดเจน (tag not found) ไม่ใช่ 500

## Scenario 14 — contact merge × tags (ทำเสร็จ 2026-07-17, E2E ผ่าน 58/58)
- **ขั้นตอน:** contact A มี tag X, contact B มี tag Y → merge A←B
- **คาดหวัง:**
  - ทุก conversation ของทั้ง A และ B แสดง X,Y รวมกัน (union, dedup)
  - assign tag ที่ sibling มีอยู่แล้ว → no-op เงียบ ๆ (200 ไม่ใช่ 201)
  - filter รายการแชทด้วย Y → conversation ของ A ติดมาด้วย
  - remove Y ที่ฝั่ง A → หายที่ฝั่ง B ด้วย (กวาดทั้งกลุ่ม — ไม่งั้น tag เด้งกลับ)
  - cap 10 นับ DISTINCT tag รวมทั้งกลุ่ม — ตัวที่ 11 → 409
  - unmerge → ต่างคนต่างเก็บ tag ที่ stamp บนตัวเอง, ไม่ต้อง migrate ข้อมูล
  - realtime: assign/remove ยิง event ถึงทุก member ในกลุ่ม (จอที่ดู sibling อัปเดตทันที)
- **จุดโชว์:** modal ใน web สัญญาไว้แล้วว่า "tags will be shared across the group" — ตอนนี้ backend ทำจริงตามนั้นแล้ว
- Demo script อัปเดตแล้ว: `scripts/test-contact-tags.sh` มี section `MERGE × TAGS` (ทั้งไฟล์ 58 checks)
