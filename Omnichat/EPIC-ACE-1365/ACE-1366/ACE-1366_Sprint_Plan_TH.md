# แผนการทำงาน EPIC-ACE-1365 (Sprint Plan)
**เรื่องที่โฟกัส (Focus Story)**: ACE-1366 (หน้าต่างแสดงโปรไฟล์ลูกค้า / Contact Profile)

## 1. เป้าหมายของ Sprint (Sprint Goals)
- พัฒนาหน้าจอฝั่งขวาของ Inbox แชทให้ครอบคลุมการแสดงผลข้อมูลแบบเบ็ดเสร็จ (ภาพรวม, ช่องทางติดต่อ, เหตุการณ์สำคัญ, และประวัติการสนทนา) 
- พัฒนา API พื้นฐาน (`GET /contacts/:id/context`) ที่จำเป็นสำหรับการรับส่งข้อมูลตัวใหม่
- จัดการ UI เพื่อให้รองรับสถานการณ์โหลดช้า หรือข้อมูลไม่ครบ (Graceful degradation) ให้ใช้งานได้ไหลลื่นขณะเอเจนต์คุยหลายแชตพร้อมกัน

## 2. ขอบเขตงาน (Scope of Stories)
- **ACE-1366 (STORY-CTX-01)**: แผนผังข้อมูลติดต่อลูกค้า (ประกอบด้วย Panel ข้อมูลส่วนตัว + ไทม์ไลน์ Key Events + ประวัติล่าสุด)

## 3. ความเสี่ยงและการอ้างอิงของงานเทคนิค (Technical Dependencies & Risk Assessment)
- **ต้องพึ่งพา (Dependency 1)**: ข้อมูลเบื้องหลังของ `Contact` และ `Conversation` ต้องมาจากการเก็บดาต้าโครงสร้าง A1 ได้ถูกต้อง
- **ต้องพึ่งพา (Dependency 2)**: แถบกิจกรรมต้องใช้ผลลัพธ์จากการ Assignment ของผู้ดูแล รวมถึงลอจิกจับเวลาจาก SLA 
- **ความเสี่ยงในระดับต่ำ (Low Risk)**: การรวบรวม Event 5 รูปแบบล่าสุดอาจจะทำให้ Queries นาน (เมื่อลูกค้าคนหนึ่งมีข้อมูลที่เยอะ) -> **วิธีลดความเสี่ยง**: สร้างคำสั่ง Limit โควต้าการดึงข้อมูลที่ฐานข้อมูล 5 รายการ พร้อมเพิ่ม Index ไว้ที่ช่อง `created_at`
- **ความเสี่ยงระดับกลาง (Medium Risk)**: ตอนที่คุณคลิกเปลี่ยนที่ Recent Conversation อาจเกิดปัญหาทำข้อความร่างปัจจุบันโดนลบทิ้ง -> **วิธีลดความเสี่ยง**: ต้องวางระบบแคช หรือ Redux เพื่อให้ตัวแชตปัจจุบันยังสามารถแสดงข้อความที่เคยใส่เอาไว้ได้ขณะสลับกลับมาใช้งาน

## 4. ลำดับกระบวนการทำงานที่แนะนำ (Recommended Execution Sequence)
1. **[ฝั่ง Backend] การสร้างฐานข้อมูลและกำหนด API**: เขียน Schema คำสั่ง Prisma และจัดทำ API  `GET /contacts/:id/context` (รวมถึงสร้าง Mockup จำลองข้อมูลส่วน Commerce/Note)
2. **[ฝั่ง Frontend] การเซ็ตอัพ Mock & โครงสร้างหน้าเว็บ**: พัฒนา `ContactProfilePanel` ฝั่งหน้าบ้าน (UI) ให้มี Skeleton หลอกการโหลดหน้าจอเอาไว้ ก่อนนำไปเชื่อมต่อกับ Mockup API ใหม่
3. **[ฝั่ง Frontend] ดำเนินการจัดทำปุ่มกดยิบย่อย**: เพิ่มกิจกรรมลงใน Mini-Activity Feed, เพิ่มป้าย Badge, และ Placeholder
4. **[ฝั่ง Frontend] เชื่อมหน้าเว็บและ Linkages**: เซ็ตคำสั่งสั่งการเชื่อมโยงต่างๆ ระหว่างกดคลิก "การสนทนาล่าสุด / Recent Conversation" 
5. **[ฝั่ง QA] การทดสอบระบบ (E2E Testing)**: ไล่ลำดับตั้งแต่ TC-01 จนถึง TC-05 โดยเฉพาะอย่างยิ่งจุดเชื่อมโยงข้อมูล หรือเช็คระบบแคชDraftเมื่อกดย้ายหน้าแชท
