# Subtask 3: จัดการประเภท Error ฝั่ง Instagram (Error Handling)

**สถานะ**: TO DO

## รายละเอียด (Description)
พัฒนาระบบดักจับ Error พิเศษต่างๆ ควบคู่หน้างานไปกับโค้ดส่งข้อความออกของ Instagram โดยสคริปต์ต้องสามารถตีความความล้มเหลวจาก Meta API ออกมาประจักษ์เป็นภาษามนุษย์หรือรหัสที่ระบบเข้าใจได้

## ขั้นตอนการทำงาน (Implementation Details)
1. **ไฟล์เป้าหมาย**: `apps/omnichat-service/src/channel-accounts/strategies/instagram.strategy.ts` (ใต้โครงสร้าง catch block เวลายิง HTTP)
2. **สถานการณ์ปัจจุบัน**: ไม่มีโค้ดส่วนนี้เลย เพราะรอคอยให้ทำ Subtask 2 ให้เสร็จก่อน
3. **สิ่งที่ต้องทำ (Action Required)**: 
   - คอยแยกประเภทก้อน JSON Exception ไม่ว่าสาเหตุจะมาจาก Token ตาย, กด Permission สิทธิ์ขาดหายตอนผูกบัญชี, หรือหน้าต่างการแชทเกิน 24 ชั่วโมงตอบกลับไม่ได้แล้ว
   - จับใส่ตัวแปร `isSuccess: false` พร้อมอธิบายตัวอักษรระบุความผิดพลาด (เช่น "นอกเหนือกฎการแชท 24 ชั่วโมง") ใส่ลงไปใน `errorMessage` 
   - ระบบกลางจะเอาแฟลกพวกนี้ไปแปะตราประทับขึ้นโชว์แดงใน UI ของทีมบริการลูกค้าว่าล้มเหลวเพราะอะไร
